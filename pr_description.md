🎯 **What:** Fixed a path traversal vulnerability (Zip Slip) in `GithubDownloader::download_and_extract` where maliciously crafted archive entries containing `../` or absolute paths could overwrite files outside the intended destination directory.
⚠️ **Risk:** Critical. If left unfixed, an attacker could provide a malicious repository archive that, upon download and extraction by Rustodian, overwrites arbitrary files on the system with the permissions of the user running Rustodian.
🛡️ **Solution:** Added path validation to explicitly check all path components of the archive entry before extraction. The path is now strictly required to consist only of `Normal` and `CurDir` (`.`) components, rejecting any `ParentDir` (`..`), `RootDir` (`/`), or `Prefix` (e.g. `C:`) components to ensure extraction stays within bounds.
