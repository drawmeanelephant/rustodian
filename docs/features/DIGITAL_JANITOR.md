# Digital Janitor

The Digital Janitor is an autonomous workspace cruft purger. It inspects tracked projects for bloated build artifacts and temporary directories, calculates reclaimable bytes, and optionally purges them to reclaim disk space.

## Cruft Targets

The Janitor automatically targets specific well-known artifact directories that are generally safe to remove, as they can be easily reconstructed by standard build tools.

| Target Directory | Description | Why it's safe to delete |
| --- | --- | --- |
| `target` | Rust build directory | Rebuilt on next `cargo build` |
| `node_modules` | Node.js / JavaScript packages | Reinstalled on next `npm install` / `yarn` |
| `.venv` | Python virtual environment | Can be recreated via `python -m venv .venv` and `pip install` |
| `.gopath` | Go workspace (Rustodian-isolated) | Dependencies fetched again by `go build` / `go mod download` |
| `.next` | Next.js build output | Regenerated on `next build` / `next dev` |
| `dist` | Generic build output | Rebuilt by the project's build tool |
| `build` | Generic build output | Rebuilt by the project's build tool |
| `__pycache__` | Python bytecode cache | Automatically regenerated by Python at runtime |

## Dry-run vs Purge

The Digital Janitor operates in two modes: dry-run (inspection only) and purge (actual deletion).

### Dry-Run Mode
By default, or when explicitly requesting a dry-run, the Janitor will only inspect the project and report what *would* be deleted and how much space *would* be reclaimed.

```bash
# Dry-run inspection
rustodian janitor my-project --dry-run
```

### Purge Mode
When the `--dry-run` flag is omitted, the Janitor executes the actual deletion of the discovered cruft directories.

```bash
# Actual purge operation
rustodian janitor my-project
```

## Calculation of Reclaimed Space

The space reclaimed (`bytes_reclaimed`) is calculated using a recursive directory walk (`dirsize`). It visits all nested files and subdirectories, summing the file sizes.

**Performance Note:** On extremely deep or file-heavy directories like large `node_modules` trees, this recursive calculation may take a noticeable amount of time due to the high volume of I/O operations required to stat each file.

## Auditing and Logs

Every purge operation is fully auditable. If directories are successfully removed (not in a dry-run), the Janitor logs the event to the `project_logs` database via `Custodian::store.save_log()`.

The log entry records the command (`janitor:clean`), the targets removed, and the total space reclaimed.

You can query this history using the CLI:
```bash
rustodian logs my-project
```

## Worked Example

Suppose you have a project with a stale Rust `target/` directory taking up 500 MB.

**Dry-run output:**
```
$ rustodian janitor example-rust-app --dry-run
Digital Janitor [Dry Run]
Found: ["target"]
Would reclaim: 524288000 bytes (500.00 MB)
No files were deleted.
```

**Actual Purge output:**
```
$ rustodian janitor example-rust-app
Digital Janitor
Purging: ["target"]
Reclaimed: 524288000 bytes (500.00 MB)
Operation logged to project history.
```
